⚠️ 重要安全声明

本页面仅为教育和研究目的设计，旨在帮助开发者了解HTML相关的安全漏洞及其防御方法。请勿将这些技术用于非法目的。所有攻击示例仅在本地环境中模拟，不会对真实系统造成危害。

场景1: 反射型XSS攻击 简单

攻击者通过URL参数注入恶意脚本，当页面显示参数值时执行攻击代码。

示例攻击载荷: <script>alert('XSS攻击成功!')</script> <img src="x" onerror="alert('通过onerror执行XSS')"> <svg onload="alert('SVG XSS')"></svg>
防御方法: 对用户输入进行HTML编码，使用innerText而非innerHTML，设置Content-Security-Policy头，使用框架的自动转义功能。

场景2: HTML注入攻击 简单

用户输入被直接插入到DOM中，导致HTML结构被篡改。

示例攻击载荷: <h2 style="color:red">网站已被黑</h2> <iframe src="http://恶意网站.com" width="0" height="0"></iframe> <div onclick="alert('点击劫持')" style="position:fixed;top:0;left:0;width:100%;height:100%;background:transparent;"></div>
防御方法: 严格验证用户输入，使用白名单过滤允许的HTML标签，对特殊字符进行转义，避免使用innerHTML直接插入用户内容。

场景3: 点击劫持攻击 中等

攻击者使用透明iframe覆盖在正常按钮上，诱导用户点击恶意内容。

<!-- 恶意网站代码示例 --> <style> iframe { position: absolute; opacity: 0.1; z-index: 9999; top: 100px; left: 100px; } </style> <iframe src="https://真实银行网站.com/转账页面"></iframe> <button style="position:absolute;top:105px;left:105px;z-index:10000;"> 点击抽奖! </button>
防御方法: 使用X-Frame-Options头部防止页面被嵌入iframe，或使用CSP的frame-ancestors指令，JavaScript防御代码检查是否在iframe中。

场景4: 开放重定向攻击 中等

利用网站的重定向功能，将用户导向恶意网站。

// 漏洞代码示例 function redirect() { const url = new URLSearchParams(window.location.search).get('redirect'); if(url) { window.location.href = url; // 危险: 未验证重定向目标 } } // 恶意URL示例 https://example.com/login?redirect=https://钓鱼网站.com
防御方法: 使用白名单验证重定向目标，避免使用用户输入直接重定向，使用中间页面警告用户即将离开本站，对重定向URL进行签名和验证。

场景5: 基于DOM的漏洞 困难

客户端JavaScript不安全地处理用户可控数据导致的漏洞。

// 漏洞代码示例 let userData = JSON.parse(decodeURIComponent(window.location.hash.substring(1))); document.getElementById("profile").innerHTML = userData.name; // 攻击载荷 https://example.com/#{"name":"<img src=x onerror='stealCookies()'>"}
防御方法: 避免使用innerHTML插入用户数据，使用textContent代替，对来自URL hash/fragment的数据进行验证和清理，实现严格的CSP策略。
